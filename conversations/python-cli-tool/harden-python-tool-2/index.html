Thank you. Can you tell me what we would need to do in order for the tool to be installable with "uv tool install stack"?

The good news is that the package is already almost there structurally. The key requirement for uv tool install is that the package has [project.scripts] entry points — and you already have:

[project.scripts]
stack = "stack:main.cli"

So the remaining question is where uv fetches the package from:

Already works today (from local path or git)

# From the repo directory:
uv tool install .

# From the git repo:
uv tool install --from git+https://github.com/bozemanpass/stack stack

No changes needed for these — they should work right now with your current pyproject.toml.

For uv tool install stack (from PyPI)

This requires publishing the package to PyPI, which means:

  1. Pick a unique package name — stack is almost certainly taken on PyPI. You'd need something like laconic-stack or bpi-stack. Update the name field in pyproject.toml accordingly. (The [project.scripts] entry can still expose a stack command regardless of package name.)

  2. Publish — You can do this with: bash uv build uv publish --token <your-pypi-token>

  3. Then users install with: bash uv tool install bpi-stack # whatever the published name is

Summary

Install method What's needed
uv tool install . Nothing — works now
uv tool install --from git+... Nothing — works now
uv tool install <name> Publish to PyPI with a unique package name

Given that you're currently distributing via shiv zipapps from GitHub releases, the git URL form might be the most practical next step without changing your release workflow.
